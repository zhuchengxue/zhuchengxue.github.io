# 学语思日常运维

## 平时只做两件事

第一，在 Dropbox 的 Obsidian 写作库中写文章。第二，双击“发布文章”，选中它并同步。

发布器会依次完成：获取 GitHub 最新版本、读取 Dropbox 原稿、整理图片、提交并推送、触发 GitHub Pages 构建；公众号已连接时，还会创建后台草稿。成功后程序自动退出。

## 文章如何判断是更新还是新建

发布器按标题匹配已有博客文章。标题相同会更新原文并保留原网址；标题不同会创建新网址。因此已经上线的文章不要随意改标题。确实需要改标题时，先在博客仓库中同步修改标题，或请 Codex 协助迁移。

## 公众号连接

在发布器中展开“本机设置”，填写公众号后台开发者 AppID 和 AppSecret。公众号还需要具备草稿接口权限，并把当前网络出口 IP 加入后台白名单。

发布器只创建草稿，不会群发。请在公众号后台预览、修改封面并手动群发。

## 常见问题

### 博客没有立刻更新

GitHub Pages 通常需要一两分钟。到 [GitHub Actions](https://github.com/zhuchengxue/zhuchengxue.github.io/actions) 查看最新任务；绿色表示发布完成。

### GitHub 推送失败

先打开 GitHub Desktop，确认已登录并能看到该仓库，再重新同步。若另一台电脑刚发布过文章，发布器会先拉取最新版本。

### Dropbox 文章找不到

确认 Dropbox 已完成同步，并在 Obsidian 中打开正确的“公众号文章”Vault。也可以在“本机设置”中直接填写写作库路径。

### 公众号草稿失败

检查 AppID、AppSecret、接口权限和 IP 白名单。博客推送与公众号草稿是两个外部步骤，界面会明确显示失败位置；修复后可以再次同步同一篇文章。

### 图片找不到

图片应位于文章同目录、Vault 根目录、`attachments/` 或 `附件/`。Obsidian Wiki 图片和普通 Markdown 图片都支持。

## 高级维护

项目诊断仍可运行 `npm run doctor -- --online`，完整本地构建可运行 `npm run build`。这些只用于开发或排障，不属于日常写作流程。
