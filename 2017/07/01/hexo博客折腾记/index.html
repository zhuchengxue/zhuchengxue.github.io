hexo博客折腾小记

发表于 2017-07-01 | | 本文总阅读量 次

在很早的时候，我就觉得那些有网站的人好厉害，仅仅是写写博客的那种以作者为域名的网站，都让人艳羡不已。一直想要有一个自己的个人独立博客网站，可以是直接用自己名字，但随着后来的了解，发现想要架网站至少需要：

  1. 一个域名，需要购买；
  2. 一个主机服务器空间，来存放自己的网站；
  3. 网站框架
  4. 前端
  5. 数据库
  6. 版本控制
  7. ……

对一个外行来说，这些词都是有字天书，我折腾过网络框架，因为之前也有一个网站“wordpress”，也能够实现这个功能，但没有深入了解。很不好搞懂，然后就放着不去管了~

机缘巧合，最近在玩linux系统，纯命令行的操作大大滴改变我对系统的认知，真正从一个菜鸟开始折腾，从基本的命令行开始，期间知道了vim，subline text等编辑器，学习git（发现这个真是一个神器，但是学习曲线也比较高），玩过html、css等，也就是在摸索git 的时候发现可以用hexo这个静态博客托管github这种形式建立自己的独立博客，然后再花点小钱在万网买了个域名，一个有专属域名的技术博客网站搞定。

踩过的坑得到的一点教训

这个网络教程一大堆，不论是linux、window还是mac都有，而且也非常的详细，那些傻瓜式的教程已足够简单，所以重复写怎么“造轮子”，这明显没啥意义。但有几个需要提醒：

  1. 虽然是傻瓜的教程，但按照步骤下来可能哪里就不对了；就像之前设置proxy的一样；
  2. node.js不要选择最新版本的，容易出毛病，我选的是4.2.3；
  3. 换一台电脑的时候，如果站点和主题设置太多了，会有很多的问题；
  4. 比如空格很重要；
  5. 有时候网站没有更改，很可能是没有清楚缓冲
    hexo clean
  6. 可以同时托管到国内的代码托管网站比如coding；
  7. 设置网站亦或是主题最好是看官方的文档，不明白的地方在问google，效率应该会高很多；
  8. 一般遇到的问题，网络其实已经早就有答案了，关键是看我们怎么快速找到它，然后把它理解了。
  9. 出错代码直接google很快就可以找到答案了
  10. 买域名的话，也有很多选择，国内域名需要实名，自己斟酌。

为什么要开这个博客？

其实微信公众号这个也是博客的一种，折腾一个已经够呛，搞两个排版，岂不是麻烦得很？主要有以下几个考量：

  1. 这个是我的网站，只要愿意折腾，可以完全的自定义自己想要的样式；
  2. 公众号的限制注定一天只能发一篇文，这个有点坑，而且排版超级麻烦（不支持markdown，写过的人都懂）；虽然有个插件可以用。
  3. 所谓的微信公众号所有权并不是个人的，是微信的，也就是现在写公号的人其实就类似租了一个地卖自己的文，文章稍微出点格，微信就会删文，如果没有备份那就真的没了；要知道每一篇文章看的人可能也就不到5min就看完了，可写的人可能花了两个多小时才敲打润色出来；
  4. 当然也有一些网站推出了博客的服务，作者需要做的也就是只是写写字就好了，但总有些限制；
  5. 因为hexo完全支持markdown，配合typora，简直就是写作最佳拍档。这样就可以专注于文字思考本身，写完排版也就搞定了。

要写什么？

公众号叫做“学语思”，我的名字有个“学”字，学本身又有学习的意思，语是说话表达的意思，思也就是思考的，连起来也就是我的写这个公众号的目的—记录我的学习思考心得。

网站的编辑，其实还有很多需要学习体会，现在也只是一个雏形，但是基本够用，所以也就是作为微信公众号的备份吧。

当然以后洗完能够写一些技术类的文章，因为在我自身的角度来看，我看别人的博客的时候，是非常想看懂别人的“干货分享”，当了这么多年的伸手党，如果能够分享一点自己的摸索心得，为后来者点灯，也是一件有意义的事情。

最后

搭建这个博客花费了我接近一周的时间，一个不意外的惊喜就是—我真的拥有自己域名的独立博客了：）

我的微信公众号，扫码一起玩！
